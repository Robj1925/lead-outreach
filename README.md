# 📱 Lead Cold Outreach SMS Sender

Reads businesses from `leads.csv` (generated by the Google Maps scraper) and sends a personalized cold outreach text via iMessage using the `imsg` CLI. Sends are staggered with a random **1–30 minute delay** between each one, and the script enforces **business hours (9 AM – 6 PM)**.

---

## Requirements

- macOS with Messages.app signed in
- [`imsg` CLI](https://imsg.to) installed: `brew install steipete/tap/imsg`
- Node.js v14+

---

## Usage

```bash
# Preview all messages without sending (ALWAYS do this first!)
node outreach.js --dry-run

# Send to just the first lead (good for testing)
node outreach.js --limit 1

# Send to first 5 leads
node outreach.js --limit 5

# Send to all leads
node outreach.js
```

---

## How It Works

1. Reads `/Users/robby/Documents/Projects/google-maps-scraper/leads.csv`
2. Skips rows with no valid phone number or already-contacted numbers
3. Enforces business hours (9 AM – 6 PM local time)
4. Sends each lead a personalized message via `imsg send`
5. Waits a random 1–30 minute delay before the next send
6. Logs every send (success or failure) to `outreach-log.json`

## 🔌 100% Free Offline Carrier Lookup

To avoid sending outreach texts to landlines (which fail and waste outreach resources) without hitting paid API limits or requiring active API keys, this project supports an **offline-capable phone carrier prefix database**.

### How to Compile the Database:
Compile the database directly from the official, daily-updated **NANPA Central Office Code assignment registry** by running:
```bash
python3 scripts/download_carrier_db.py
```
This downloads the official zip, parses utilize codes, filters wireless carriers, and compiles a highly compressed local lookup file `us_mobile_prefixes.json` (approx. 2MB) that loads into memory in milliseconds.

### How the Lookup Works:
1. **Primary API Check**: If `NUMVERIFY_API_KEY` and `ABSTRACT_API_KEY` are configured in `.env`, the script will query them.
2. **Automatic Offline Fallback**: If keys are missing, invalid, or their free limits/quotas are exhausted, the script automatically falls back to the local `us_mobile_prefixes.json` database.
3. **Offline-First Mode**: If no API keys are present in `.env` but the database is compiled, the script performs 100% of the carrier lookups locally and instantly with **zero network requests**!

## Safety Features

- **Dry-run mode** — preview before sending
- **Business hours enforcement** — will not send before 9 AM or after 6 PM
- **Deduplication** — skips numbers already in `outreach-log.json`
- **Offline Carrier Check** — instant fallback to NANPA database to skip landlines when API keys are exhausted or missing
- **--limit flag** — cap how many you send in a session
- **Full logging** — every send is recorded with timestamp and status
